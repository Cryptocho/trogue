# Autotile 3x3 (minimal) Bitmask

## 一、基础结构：3×3掩码矩阵定义
把当前瓦片自身放在**矩阵中心 [1][1]**，矩阵坐标规则（行0/1/2，列0/1/2）：
```python
# 3x3 mask 矩阵布局（直观演示）
mask_grid = [
    [左上(0,0), 上边缘(0,1), 右上(0,2)],
    [左边缘(1,0), 中心(1,1), 右边缘(1,2)],
    [左下(2,0), 下边缘(2,1), 右下(2,2)]
]
```
9个位置分为三类：
1. **中心位 (1,1)**：必须设为 `On`，否则该瓦片永远不会被引擎选用；所有可用瓦片强制中心On
2. **四边边缘位**：`(0,1)上 / (1,0)左 / (1,2)右 / (2,1)下`
3. **四角对角位**：`(0,0)左上 / (0,2)右上 / (2,0)左下 / (2,2)右下`

### 三种位状态（编辑器操作）
- **On（红）**：该方向**必须存在同Autotile瓦片**，不满足则匹配失败
- **Off（空白）**：该方向**必须没有同Autotile瓦片**，存在则匹配失败
- **Ignore（灰，Shift+左键）**：匹配时完全跳过该位，不做校验

## 二、核心匹配规则（Minimal 独有约束，和完整3x3区分）
### 1. 边缘位规则（优先级最高）
- 边缘位 = On：共享这条边的相邻格子**必须填充**同Autotile
- 边缘位 = Off：共享这条边的相邻格子**必须为空**
- Ignore：不判断这条边是否有瓦片

### 2. 四角位联动约束（Minimal关键简化逻辑）
四角位**不能孤立生效**：
> 若任意角位设为 On，则该角相邻的**两条边缘位必须同时为 On**，否则该掩码逻辑无效、永远匹配不到场景

例：`(0,0)左上 = On` → 必须同时满足 `(0,1)上=On` 且 `(1,0)左=On`
- 原因：Minimal模式对角瓦片仅作为**边角修饰**，依赖上下左右四条主边是否连通；完整3x3无此限制。

### 3. 引擎匹配完整流程
1. 读取TileMap当前格子，扫描周围8邻域，生成**场景掩码（运行时状态）**
2. 遍历Autotile内所有瓦片的预设Bitmask：
   - 对比所有 `On/Off` 位，必须**全部严格吻合**；Ignore位直接跳过
   - 中心位必须On，否则直接跳过该瓦片
3. 多瓦片匹配成功：按瓦片`Priority`权重随机选；无匹配则用Autotile的Icon兜底瓦片

## 三、矩阵示例：各类常用瓦片掩码完整演示
约定矩阵值标注：`1=On，0=Off，X=Ignore`；全部示例中心`(1,1)=1`（强制）

### 示例1：孤立单格瓦片（四周无邻居）
需求：上下左右四角全都没有同Autotile瓦片
```
isolated = [
    [0, 0, 0],
    [0, 1, 0],
    [0, 0, 0]
]
```
规则校验：四边边缘全Off，四角自然全Off，无冲突。

### 示例2：横向长条中间瓦片（左右连通，上下空）
左右边缘On，上下边缘Off；四角全部Ignore（无需校验对角）
```
horizontal_mid = [
    [X, 0, X],
    [1, 1, 1],
    [X, 0, X]
]
```
运行时匹配条件：左边有瓦片、右边有瓦片；上边、下边无瓦片。

### 示例3：单端端头瓦片（仅右侧连通，其余全空）
右边缘On，上/左/下边缘Off；四角全Off
```
right_end = [
    [0, 0, 0],
    [0, 1, 1],
    [0, 0, 0]
]
```
匹配：仅右边存在瓦片，其余方向全空。

### 示例4：L型拐角（右上拐角：上边+右边连通）
1. 边缘：上(0,1)=On、右(1,2)=On；左/下=Off
2. 四角：右上(0,2)=On；其余四角Off（满足约束：角On的两条相邻边都是On）
```
corner_top_right = [
    [0, 1, 1],
    [0, 1, 1],
    [0, 0, 0]
]
```
约束校验：角(0,2)=On，相邻边(0,1)上、(1,2)右均为On → 合法掩码。

### 示例5：十字交叉瓦片（上下左右四边全连通）
四边边缘全部On；四角全部Ignore（对角不做限制）
```
cross = [
    [X, 1, X],
    [1, 1, 1],
    [X, 1, X]
]
```
匹配条件：上、下、左、右四个相邻格子都存在同Autotile瓦片；对角有无不影响。

### 示例6：T型岔路口（上、左、右连通，下方为空）
上/左/右边缘On，下边缘Off；四角Ignore
```
t_junction = [
    [X, 1, X],
    [1, 1, 1],
    [X, 0, X]
]
```

### 示例7：复杂多拐角（全包围内部地砖，四边四角全连通）
四边边缘On，四个角全部On（每个角相邻两条边均为On，符合约束）
```
full_fill = [
    [1, 1, 1],
    [1, 1, 1],
    [1, 1, 1]
]
```

## 四、Minimal vs 完整3x3 核心区别（为什么Minimal更常用）
1. **组合数量**
   - 3x3 minimal：有效合法掩码仅 **47种**，做地形/墙体仅需47张瓦片即可全覆盖所有连通形态
   - Full 3x3：2⁹=512种组合，需256张瓦片，对角位可独立设置，无边缘联动约束，适合精细像素分层
2. **对角位逻辑**
   - Minimal：角位依赖相邻两条边必须同时On，对角不能单独生效
   - Full 3x3：四角、四边完全独立，可单独设置对角On而相邻边Off，无约束
3. **适用场景**
   - Minimal：草地、墙体、地面地砖、俯视地牢（90%2D游戏通用）
   - Full 3x3：斜45度透视、多层叠加瓦片、需要单独控制对角衔接的特殊美术

## 五、常见错误掩码案例（Minimal模式下失效）
### 错误1：中心位=0
```
invalid_center = [
    [0,0,0],
    [0,0,0],
    [0,0,0]
]
```
后果：引擎永远不会匹配此瓦片，文档强制中心必须On。

### 错误2：角位On，但相邻一条边Off（违反Minimal约束）
```
invalid_corner = [
    [0, 0, 1], # 右上(0,2)=On；上边(0,1)=Off
    [0, 1, 1], # 右边(1,2)=On
    [0, 0, 0]
]
```
问题：角(0,2)相邻的上边是Off，掩码逻辑非法，地图绘制时不会触发该瓦片。

## 六、运行时场景掩码生成示例（直观理解匹配）
假设TileMap里当前格子坐标`(5,5)`，邻域状态：
- 上方`(5,4)`有瓦片、右方`(6,5)`有瓦片；左、下、四角全部为空
生成场景运行掩码矩阵：
```
scene_mask = [
    [0, 1, 0],
    [0, 1, 1],
    [0, 0, 0]
]
```
引擎要自动匹配上面**右上拐角corner_top_right**的预设bitmask，渲染对应拐角瓦片。
