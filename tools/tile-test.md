# tile.py 三态位掩码测试文档

## 测试环境

- Python 3.8+
- tkinter (随 Python 分发)
- Pillow (PIL)

运行方式：

```bash
cd /mnt/WD/workspace/trogue
python tools/tile.py
```

## 功能测试用例

### TC-01: 三态切换（编辑器内）

1. 选择一个已标记的 tile，打开位掩码编辑器
2. **左键点击** 一个 Off(0) 单元格 → 应变为黄色(On=1)
3. **再次左键点击** 同一单元格 → 应变为透明(Off=0)
4. **Shift+左键点击** 一个 Off(0) 单元格 → 应变为灰色带斜线(Ignore=2)
5. **再次 Shift+左键点击** 同一单元格 → 应变为透明(Off=0)
6. **左键点击** 一个 Ignore(2) 单元格 → 应变为黄色(On=1)
7. **右键点击** 任意单元格 → 应强制变为透明(Off=0)

**预期结果**：三态切换行为与计划一致。

### TC-02: 拖拽绘制

1. 在编辑器内，左键拖拽多个单元格
2. 起始单元格状态应被复制到所有经过的单元格
3. 如果起始单元格是 Ignore(2)，拖拽路径上的单元格都应为 Ignore
4. 如果起始单元格是 On(1)，拖拽路径上的单元格都应为 On

**预期结果**：拖拽行为与单次点击逻辑一致。

### TC-03: 中心单元格自动强制为 On

1. 在编辑器中，将中心单元格 `(1,1)` 设为 Off(0) 或 Ignore(2)
2. 点击"确定"保存
3. 重新打开该 tile 的位掩码编辑器
4. 检查中心单元格 `(1,1)` 是否为 On(1)

**预期结果**：中心单元格始终为 On，即使用户在保存前将其设为其他值。

### TC-04: Minimal 约束警告

1. 在编辑器中设置以下位掩码：
   ```
   1 0 0
   0 1 0
   0 0 0
   ```
   （左上角 On，但其上边和左边为 Off）
2. 点击"确定"
3. 应弹出警告对话框，内容包含"违反 Minimal 约束"
4. 点击"否" → 对话框应保持打开，不保存
5. 修改位掩码满足约束后再次点击"确定" → 应正常保存关闭

**预期结果**：违反约束时警告，允许用户强制保存或取消。

### TC-05: 约束自动修复中心

1. 在编辑器中设置：
   ```
   1 1 1
   1 0 1
   1 1 1
   ```
2. 点击"确定"
3. 重新打开，检查中心是否为 On

**预期结果**：中心自动修复为 On。

### TC-06: 格式刷复制 Ignore 状态

1. 选择一个带有 Ignore(2) 单元格的 tile
2. 点击"格式刷"按钮
3. 点击另一个 tile
4. 打开该 tile 的位掩码编辑器
5. 检查被格式刷覆盖的单元格是否为 Ignore(2)

**预期结果**：格式刷正确深拷贝 Ignore 状态。

### TC-07: 主视图预览渲染三态

1. 为某个 tile 设置包含 Ignore(2) 的位掩码
2. 返回主视图，确认该 tile 的 bitmask 预览中 Ignore 单元格显示为灰色块
3. 开启格式刷模式，悬停在其他 tile 上
4. 如果格式刷包含 Ignore 状态，预览应显示灰色块

**预期结果**：主视图和格式刷预览都能正确区分三态。

### TC-08: 导出 Lua 代码

1. 为若干 tile 设置包含 On(1)、Off(0)、Ignore(2) 混合的位掩码
2. 点击"导出 Lua"
3. 检查生成的 Lua 代码：
   - 顶部注释应包含 `Bitmask values: 0 = Off, 1 = On, 2 = Ignore`
   - bitmask 表格中应正确输出 0、1、2 值
   - 仅含 Off(0) 的 tile 不应输出 bitmask 字段

**预期结果**：导出格式正确，注释说明清晰。

### TC-09: 旧数据兼容性

1. 手动编辑 `state["bitmasks"]`，插入仅含 0/1 的旧格式位掩码
2. 在主视图和编辑器中打开该 tile
3. 确认显示正常（On=黄色，Off=透明）
4. 导出确认格式正确

**预期结果**：旧数据无需迁移，自动兼容。

### TC-10: 反选操作

1. 标记多个 tile 并设置位掩码
2. 点击"反选"
3. 确认标记状态反转，位掩码仅保留在新标记的 tile 中

**预期结果**：反选操作不受三态影响。
